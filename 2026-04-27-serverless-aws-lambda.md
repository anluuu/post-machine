# AWS Lambda Evolution in 2026

Lambda isn't just for quick scripts anymore. The 2026 updates are changing what it can do:

**Lambda Durable Functions**
- Complex stateful workflows with checkpointing
- Pause/resume without active compute billing
- Think long-running ETL, multi-step data processing

**Managed Instances**
- Run on dedicated EC2 infrastructure
- GPU and memory-optimized options
- Good for ML inference without container overhead

**Provisioned Mode for SQS**
- Faster scaling, higher concurrency
- Predictable performance under burst workloads
- Bridge between serverless and reserved capacity

**Who's using it:**
- 11.95% serverless market share (#3)
- 99,491 companies, 1.5M monthly active users
- 65% of AWS customers use Lambda
- Mostly small teams (1-10 employees)

**What I'm watching:**
- How Managed Instances compete with Cloud Run
- Whether Durable Functions reduce need for Workflows
- Python 3.14 adoption for AI orchestration

Lambda's becoming a full application platform, not just a helper.
